推奨事項

以下は推奨事項です。以下のものが満たされていないからといってテーマが承認されないということはあってはなりませんが、ベストプラクティスとして考えられています。

Code

テーマはコアのウィジェットを登録解除することができます。

コアの関数や機能

以下のフォームについてはコアのマークアップを利用することが推奨されます。

  • ログインフォーム
  • 検索フォーム

以下の機能を組み込むのであれば、WordPressに含まれるものを利用しなければなりません。

  • フィードリンクの自動出力:
    • add_theme_support( 'automatic-feed-links' )
  • サイドバー：
    • register_sidebar()
    • dynamic_sidebar()
    • add_action(widgets_init), register_widget() (Widgets API も見てください) – (カスタムウィジェットを利用するのであれば)
  • ナビゲーションメニュー:
    • wp_nav_menu()
    • register_nav_menu() または register_nav_menus() を functions.php で。
    • 次の関数は、ナビゲーションメニューのために、 wp_nav_menu() のコールバックとして、 has_nav_menu() の条件分岐として、あるいは類似の形でのみ使うことができます。： wp_list_pages, wp_list_categories, wp_page_menu など。
  • アイキャッチ画像 Post Thumbnails:
    • the_post_thumbnail()
    • add_theme_support('post-thumbnails') in functions.php
  • カスタムヘッダー:
    • add_theme_support('custom-header') in functions.php
  • カスタム背景:
    • add_theme_support('custom-background') in functions.php
  • エディタースタイル:
    • add_editor_style()
    • editor-style.css file

ドキュメンテーション

以下はドキュメンテーションについての推奨事項です。

  • readme.txt を含めることができます。
  • 変更に関しては changelog.txt も同梱するととてもよいです。
  • どこでもよいので、変更履歴を残しましょう。
  • WordPressが標準で行わないことを追加するのであれば、どんなものであれドキュメントしてください。
  • すべての種類のユーザーについて考えて、誰にでも使えるようにしてください。
  • 特別なことをやっている場合にはどんなことでもドキュメントしてください。
  • テンプレートやCSSに行ったことは分かりやすく言及してください。
  • 変更、テンプレートセクション、CSSスタイルにコメントを付けます。特にテンプレートをまたぐような時は。
  • テーマについてのサポートをどこで得られるのかを書いておくこともよいです。

販売、テーマ、リンク

  • Theme URI はオプションです。使うのであればテーマについての情報があるサイトであることが必須です。でもサイトであっても、コンテンツはテーマについてのことでなければならず、テストデータではいけません。
  • Author URI はオプションです。使うのであれば、開発者の個人のウェブサイトか、プロジェクト／開発についてページである必要があります。

テンプレート

  • テーマ内で使われる場合、標準のテンプレートファイルについては正しいテンプレートタグを使って呼び出す必要があります。
    • header.php (get_header()経由)
    • footer.php (get_footer()経由)
    • sidebar.php (get_sidebar()経由)
    • comments.php (comments_template()経由)
    • searchform.php (via get_search_form()経由)
  • オリジナルのテンプレートファイルを含める場合には、何をするファイルなのかドキュメントをします。

テンプレートタグ

  • テーマは、関数の引数を使って出力を変更することができます。(body_class( $class )/post_class( $class ))
  • テーマは、フィルターを使って出力を変更することができます。(body_class/post_class)
  • 対応するテンプレートを使う場合、以下を含める必要があります。
    • wp_title()
    • wp_head() – (</head>の直前)
    • body_class() – (<body>タグに入れる)
    • $content_width
    • post_class()
    • wp_link_pages()
    • paginate_comments_links(), または previous_comments_link()/next_comments_link()
    • posts_nav_link(), または previous_posts_link()/next_posts_link(), または paginate_links()
    • wp_footer() – (</body>の直前)
