テーマレビュー

必須項目

WordPress.org テーマリポジトリにあるテーマには、以下の全ての必須項目を含めなければなりません。 テーマチェックにあたり、これらのチェックは、テーマチェックプラグインを通してテーマを立ち上げなければならない点を再度確認してください。自動で全ての必須項目のチェックが行われます。

アクセシビリティ

  • テーマに 'accessibility-ready' タグが含まれる場合、アクセシビリティの必須項目をすべて含んでいる必要があります。

コード

  • PHP や JavaScript のエラーがでていない。
  • 最低でも index.php と style.css がある。
  • 有効な DOCTYPE 宣言と language_attributes を含んでいる。
  • すべてサニタイズされている。
  • テーマで使っていないフックを追加したり削除したりしていない。
  • ショートコードを含んでいない。
  • WordPress が生成するこれらの CSS クラスに対応している:
    • alignleft
    • alignright
    • wp-caption
    • wp-caption-text
    • gallery-caption
    • sticky (can be unstyled)
    • bypostauthor (can be unstyled)

コアの関数や機能

  • 可能なかぎり、 WordPress の関数や機能を使う。
  • Don’t include admin/feature pointers.
  • カスタム投稿タイプ・カスタム分類を含まない。
  • どんな WordPress の機能も、有料でないと使えない範囲を作ってはいけません。
  • 管理者ツールバーを使えなくしない。
  • テンプレートまでのパスを取得するのに、 TEMPLATEPATH ではなく get_template_directory() を使う。
  • スタイルシートまでのパスを取得するのに STYLESHEETPATH ではなく get_stylesheet_directory() を使う。
  • コンテンツを生成するのに、ハードコーディングをしない。 代わりに、適切な方法で関数のパラメーター、フィルターとアクションフックを使いましょう。例えば wp_title の値は、フィルタを使って変更しなければなりません。
  • テーマから子テーマが作れるようにする。
  • comments_template() を含める

ドキュメンテーション

  • あらゆるカスタム機能、オプションや限定事項も、（例えばメニューの構造）説明が必要です。じゅうぶんなドキュメンテーションを提供してください。

Favicons

  • ファビコンを実装する場合、標準では無効にし、ユーザーが上書きできる状態にしてください。

言語

  • 全てのテーマ中の文章は翻訳可能にしましょう。
  • style.css の中にテキストドメインを含めましょう。
  • 単一のユニークなテーマスラッグを使います。スラッグは style.css に書いた通りにします。フレームワークを使っている場合も、２つ以上スラッグを使いません。
  • テキストには何語でも使えます、でもすべてのテキストは同じ言語で書いてください。

ライセンス

  • 100% GPL および／または 100% GPL に準ずるライセンスにします。
  • 著作権とライセンスを明示します。ライセンスとライセンス uri は style.css のヘッダーに書きます。
  • フォントや画像を含む全てのリソースのライセンスを明示する。
  • 全てのコードとデザインは、あなた自身がつくったもの、または法的にあなたのものとなってるものを使ってください。よそからのコピーは認められません。

名前

  • テーマの名前に「WordPress」を含めないでください。
  • あらゆるテキストの中で “WordPress” を正しい綴りで書いてください: スペースを入れないことと、 W と P が大文字です。

オプションと設定

  • オプションは単一の配列に保存します。
  • sane defaultsを使って、標準設定のときはデータベースに書き込まない。
  • Use edit_theme_options capability for add_theme_page(), rather than rely on a role (e.g. “administrator”), or a different capability (e.g. “edit_themes”, “manage_options”) for the capability to add the settings page.
  • テーマオプションのページを使う場合、「外観」メニューのなかに追加してください。
  • テーマオプションの実装には、 Settings API またはテーマカスタマイザーのいずれかを使ってください。
  • すべてのオプション、カスタム関数、カスタム変数、カスタム定数は、テーマスラッグをプレフィクスしてください。

プラグイン

  • プラグインは同梱しないでください。テーマは、プラグインのおすすめはしてもよいですが、テーマのコードに含めないでください。
  • プラグインの領域と考えられることを、テーマでやらないでください。

スクリーンショット

  • Screenshot.png は、実際の標準設定のときのテーマにしてください。ロゴやモックアップではありません。
  • screenshot.png は 880x 660px よりも大きくしないでください。

セキュリティとプライバシー

  • ユーザーの同意なしに、自宅に電話しないでください。 詳しくはセキュリティについてをお読み下さい。
  • ユーザーのデータをとる場合は、かならずテーマオプションで設定できるようにしてください。デフォルトはかならず無効にしてください。
  • untrusted データは、データベースに入れる前に検証、サニタイズしてください。すべての untrusted データは、出力される前にエスケープしてください。（データの検証をご覧ください。）
  • テキストの入力には esc_attr() を使い、テキストエリアには esc_html() を使ってください。

販売、著者クレジット、リンク

  • テーマ販売者が WordPress.org レポジトリにテーマを登録する場合、ほかに販売しているテーマも GPL でなければなりません。
  • テーマの広告をフッターのリンクに追加する場合、１つだけにしないといけません。

スタイルシートとスクリプト

  • スクリプト、スタイルとファビコンは、ブラウザ対策のため以外ではハードコーディングしないでください。ページ読み込みが終わったあとに動くべきです。
  • 解析やトラッキングをしない。
  • 元のファイルを提供していない場合、スクリプトやその他のファイル難読化をしないでください。
  • コアに入ってるスクリプトがある場合はそちらを使って、独自のバージョンのものを使わないでください。例えば jQuery　など。
  • 全てのスクリプトとリソースは直リンクせず、同梱してください。ただし Google ライブラリーは除きます。
  • もし style.css にタグを書いた場合、テーマはそのタグが意味するような仕様になってないといけません。例えばカスタム背景やヘッダー。

テンプレート

  • テンプレートを使う場合、カスタムテンプレートファイルは get_template_part() または locate_template() を使って呼び出します。
  • 特定のものがすでに使われている場合、いくつかの テンプレート特有のこと を考慮すべきです。
  • bloginfo() と同等のものは、テンプレートタグ *_url() を使いましょう。
It's worth noting we are working to automate a lot of the above requirements.
